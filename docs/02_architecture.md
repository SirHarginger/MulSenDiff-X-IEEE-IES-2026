# System Architecture

## Title

MulSenDiff-X: category-aware descriptor-conditioned diffusion for multi-sensor industrial anomaly detection

## Core Idea

MulSenDiff-X models normal RGB appearance while conditioning the detector on structured auxiliary evidence derived from:

- infrared
- point clouds
- cross-modal alignment cues

The auxiliary branch is represented as **sensor-derived thermal-geometric descriptors**, not as a second raw-image generative stream.

## Detector Regimes

The repo supports three detector regimes:

- `Shared`
  - one shared model across all 15 categories
  - this is the headline method

- `PerCat`
  - one category-specific model per object type
  - this is the benchmark-compatible reference

- `Shared-NoCat`
  - shared training without category identity
  - this is the key ablation for testing category-aware conditioning

These are not different products. They are different operating regimes of the same MulSenDiff-X pipeline.

## End-To-End Pipeline

1. Load aligned RGB, infrared, and point-cloud inputs for one part instance.
2. Convert infrared and point cloud into:
   - spatial descriptor maps
   - support maps
   - a compact global descriptor vector
3. Feed RGB plus the structured descriptor stack into a conditional diffusion detector.
4. Score anomalies from reconstruction and denoising evidence, local descriptor support, and a robust global descriptor branch.
5. Calibrate operating decisions and localization using held-out normal data.
6. Export structured evidence for optional explanation and the inspection app.

## Conditioning Stack

The learned conditioning path is now a multimodal stack that includes:

- spatial descriptor maps
- support maps
- global descriptor vector
- category embedding in `Shared`
- no category identity in `Shared-NoCat`

This means the detector is not using IR and point cloud only as post-hoc metadata. They influence the learned conditioning and the downstream scoring path.

## Detector Scoring Path

The scoring path combines:

- RGB residual / denoising evidence
- local descriptor support
- robust global descriptor deviation

The corrected detector now uses:

- calibrated operating thresholding from held-out normal data
- robust global descriptor scoring
- global-branch gating when local evidence is weak
- localization calibration for region extraction and affected-area estimation

This is the scientific scoring path. The app reads its outputs; it does not define them.

## Training And Evaluation Protocol

The repo no longer treats the official test split as a selection surface.

Runtime protocol:

- train on `train_core_good`
- calibrate and select on `calibration_good + synthetic_validation`
- evaluate on `official_test`

This split logic is generated by the repo and documented in [docs/01_dataset.md](/home/eddy/Downloads/sirharginger/MulSenDiff-X-IEEE-IES-2026/MulSenDiff-X-IEEE-IES-2026/docs/01_dataset.md).

## Product And Explanation Layer

The app is a demo surface, not the main scientific artifact.

Current deployment story:

1. MulSenDiff-X performs detection, localisation, and calibrated severity/area estimation.
2. The detector exports a structured evidence payload.
3. trusted retrieval attaches category-aligned support from `docs/references/`.
4. Gemini turns that grounded evidence into an operator-facing explanation.

Gemini is not part of detector scoring and should not be treated as the source of anomaly decisions. In the current app, retrieval + Gemini are the intended explanation path, with detector-grounded fallback kept only for reliability.

## Paper Positioning

For publication, the intended detector story is:

- `Shared` is the headline method
- `PerCat` is the benchmark reference
- `Shared-NoCat` is the key ablation

Main-paper extras stay narrow:

- modality-visibility analysis
- descriptor robustness

Everything else, including explanation evaluation and ROC/PR curves, is secondary or appendix-only.
